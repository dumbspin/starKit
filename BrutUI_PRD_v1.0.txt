— Section 01
Executive Summary

BrutUI is an open-source, neo-brutalist React UI component library targeting portfolio developers and creative frontend engineers. It ships as both an installable npm package and a copy-paste CLI (modelled after shadcn/ui), giving developers full flexibility over how they consume it.

The library fills a clear gap in the current ecosystem: Shadcn/UI owns 'neutral and headless', Aceternity owns 'animated glassmorphism', and no well-documented, well-maintained library owns neo-brutalism. BrutUI claims that space with zero-radius corners, bold 2px borders, hard offset box-shadows, and a loud primary palette built on yellow, red, and blue.

The target audience — portfolio and creative developers — actively seeks a strong visual identity out of the box. They want their projects to look opinionated and distinctive from the first install, without spending weeks designing a custom system from scratch. BrutUI gives them that in one command.

One-line positioning
"The UI library for developers who are tired of everything looking the same."

— Section 05
Design System Specification

The design system is not a style preference — it is the product. Every decision is intentional and enforced. Deviating from these rules produces something that is no longer BrutUI.

The Three Laws of BrutUI
Law 1: The Hard Shadow
Every interactive element carries a hard, flat offset box-shadow: 4px 4px 0px #000. On hover, it collapses to 2px 2px 0px #000, creating a press-into-the-page effect. This is non-negotiable. It is the single visual signature of the library.

Law 2: Zero Radius
border-radius is 0px everywhere. No rounded corners, ever. The rawness of neo-brutalism comes from refusing to soften edges. If a component uses a radius, it is not a BrutUI component.

Law 3: Bold Borders
Every element that has a border uses a minimum of 2px solid #000. No hairlines. No subtle separators. If there is a border, it is visible from across the room.

Typography Rules
Headings use font-weight 900 (Black). Body uses 400. Interactive labels use 700 (Bold).
No proprietary font is bundled. The library defaults to system-ui, then falls back to sans-serif. Users pair it with any font — Inter, Space Grotesk, and Syne work especially well.
Recommended pairing for docs site and marketing: Space Grotesk (headings) + Inter (body).
Minimum font size: 14px for body, 12px for labels and captions.

Interaction Rules
Hover state: transform: translate(2px, 2px) + shadow collapses from md to hover. Transition: 80ms ease — fast, snappy, not smooth.
Active/pressed state: transform: translate(4px, 4px) + shadow: none. The element fully 'sinks' into the page.
Focus state: 2px solid #0066FF outline, 2px offset. Never remove focus outlines — accessibility is non-negotiable.
Disabled state: opacity 0.4, cursor: not-allowed, no shadow, no hover effect.

7. Toggle / Switch
Width 48px, height 24px. Track has 2px border. Thumb is 16px square (not rounded — brutalism rule).
On state: yellow track. Off state: gray track. Transitions at 100ms ease.
Props: checked, disabled, label, onChange, size (sm | md).

8. Alert
Variants: info (blue left border), success (green), warning (yellow), danger (red).
Left border is 4px thick — the only place a colored border appears. Background is a light tint of the variant color.
Props: variant, title, description, dismissible, icon (boolean), onDismiss.

9. Avatar
Shape: square (brutalism rule — no border-radius). Sizes: xs (24px), sm (32px), md (40px), lg (56px), xl (72px).
Fallback: initials on gray background. Image fallback: initials. Supports src, alt, fallback, size.
Group variant: overlapping avatars with black border between them.

10. Tooltip
Trigger: hover or focus. Position: top | bottom | left | right (auto-flips near viewport edge).
Styled: black background, white text, 2px border. Arrow is a square rotated 45 degrees.
Props: content, position, delay (ms), children.

11. Select
Custom dropdown — no native select. Opens a flat panel with 2px border and shadow-md below trigger.
Options support icons, descriptions, and dividers. Keyboard navigation required (arrow keys, enter, escape).
Props: options, value, onChange, placeholder, disabled, multiple.

Height variants: sm (4px), md (8px), lg (16px). Track: gray background, 2px border.

Phase 3 — Signature Components (Weeks 9–12)
These are the components that make people share screenshots. They are uniquely BrutUI — you cannot get these from Shadcn or MUI. They exist to drive discovery, word of mouth, and GitHub stars.

Glitch Text
A text component that applies a CSS glitch animation — chromatic aberration effect with pseudo-element shifting. Configurable intensity and speed. Common in portfolios and hero sections.
Props: text, intensity (low | medium | high), speed, color, trigger (hover | auto | always).

Scramble Text
Text that scrambles through random characters before resolving to the real text. Inspired by hacker UIs. Can be triggered on mount, on hover, or programmatically.
Props: text, duration, charset (alphanumeric | symbols | full), trigger, onComplete.

Marquee
Smooth horizontal scrolling ticker. Supports any ReactNode as children — not just text. Pause on hover. Bidirectional. Variable speed.
Props: speed, direction, pauseOnHover, repeat, gap, children.

Noise Overlay
A Canvas-based grain/noise texture overlay for adding texture to backgrounds without images. Adjustable grain size, opacity, and animation speed. Performance-optimised with requestAnimationFrame.
Props: opacity, grainSize, animated, blend (multiply | overlay | screen).

Cursor FX
Replaces the native cursor with a custom follower — a square (brutalism) that trails the mouse with a configurable lag. Changes size on hover over interactive elements. Toggle on/off.
Props: size, color, lag, hoverScale, enabled.

Drag Card
A card that can be freely dragged within its container. Snaps back on release (optional). Multiple drag cards create an organic, desk-like layout. Great for portfolio sections.
Props: defaultPosition, bounds, snapBack, snapBackDuration, onDragEnd, children.

Sticker
A rotatable, resizable badge/label that feels like a physical sticker. Slightly random initial rotation. Peels on hover. Four shape variants: rectangle, circle, star, arrow.
Props: label, color, shape, rotation (random by default), size, peelOnHover.

Ticker / Scoreboard
An animated number ticker. Numbers count up or down from a start value to an end value with configurable easing. Perfect for stats sections. Direction-aware — flips animation for up vs down.
Props: from, to, duration, easing, format (currency | percent | number), prefix, suffix.

packages/core — npm Package
Output formats: ESM (.mjs) and CJS (.cjs). Types: .d.ts files co-located with output.
CSS: exported as a separate brutui.css file. Users import it once at the app root: import 'brutui/styles'.
Tree-shakeable: each component is a named export. Unused components are dropped by bundler.
Peer dependencies: react ^17 | ^18, react-dom ^17 | ^18. Nothing else.
Bundle size target: < 10kb gzipped for the full library (CSS tokens drive styles, not JS).

packages/cli — Copy-Paste Tool
Command: npx brutui add [component]. Downloads the component source files directly into the user's project.
Interactive prompt if no component name given: shows a checklist of all available components.
Respects project structure: detects if user has src/ directory and places files accordingly.
Config file: brutui.json at project root stores user preferences (output directory, TypeScript preference, CSS framework).

CLI Example Interaction

TypeScript Conventions
All props interfaces are exported and named: ButtonProps, CardProps, etc.
Variant types use string literal unions: type ButtonVariant = 'primary' | 'danger' | 'ghost' | 'outline'
Every component accepts className and style as escape hatches.
forwardRef used on all interactive elements (Button, Input, Select, Textarea) for ref forwarding.

Accessibility Requirements
All interactive components meet WCAG 2.1 AA minimum. This is non-negotiable and part of every component's acceptance criteria.
Focus management: focus trapping in Modal and Drawer, focus restoration on close.
ARIA: all components use correct roles, aria-labels, aria-expanded, aria-controls as appropriate.
Keyboard navigation: all interactive components are fully keyboard operable. Tab, Enter, Space, Escape, arrow keys as per ARIA patterns.
Color contrast: all default color combinations meet 4.5:1 contrast ratio. Yellow (#FFE600) on black (#000) = 19.6:1.


— Section 08
Distribution Model

BrutUI offers both distribution methods simultaneously. The package is designed so both methods produce identical output — users get the exact same component regardless of how they install it.

Method 1: npm Package
Installation
Install command
npm install brutui

Usage
Usage
import { Button, Card, Badge } from 'brutui';
import 'brutui/styles';

export default function App() {
  return <Button variant="primary">Click me</Button>;
}

Best for: developers who want to get started in under 60 seconds and don't need to customise internals.
Updates: standard npm update workflow. Follows semver strictly.
Bundle impact: tree-shaking removes unused components. Only imported components appear in final bundle.

Method 2: CLI Copy-Paste
Installation
CLI command
npx brutui add button
npx brutui add card badge input  # multiple at once
npx brutui add --all             # install everything

Best for: developers who want full source ownership and the ability to customise every detail.
Each component is copied as a standalone .tsx + .css file into the project.
No ongoing dependency — the user owns the code after installation.
The CLI checks for updates and notifies users when a newer version of a component is available.

Versioning Strategy
Semantic versioning (semver) strictly followed: major.minor.patch.
Breaking changes (new Law violation, token renames, prop removals) = major version bump.
New components, new variants, new props = minor version bump.
Bug fixes, style tweaks = patch version bump.
Changesets used for automated changelog generation and npm publish workflow.
